Vagrant freezes on 'SSH auth method: private key'

By Sam
March 7, 2021

If the connection times out after you run vagrant up  and it takes too long and times out after the following.

default: SSH address: 127.0.0.1:2222
default: SSH username: vagrant

One of the reasons this could happen is because virtualization is disabled in the bios. To check if if virtualization is enabled on a Linux system, run sudo kvm-ok.  The output should inform you if it is disabled. 

To enable it go into the bios and under CPU or processor enable virtualization. If virtualization was causing it, now you should be able to run vagrant.
